# 都市奇幻章节写作 Skill

## 适用场景

当用户提出以下需求时，默认启用本 skill：
- 按大纲写某一章或连续多章
- 不回读历史正文，直接按摘要填充，可参考前文摘要保持连贯
- 写完后按卷和章保存文件

## 角色边界（硬性）

- 本 skill 只负责：读取 `OUTLINE.md` 并填充章节正文
- 本 skill 不负责：重做大纲、改世界观底层设定
- 若大纲不足以支持写作，先提出最小澄清问题，不自行扩写总设定

## 输入依赖（硬性）

写作前只允许读取：
- `workspace/novels/<书名>/OUTLINE.md`
- `OUTLINE.md` 中目标章节之前的章节摘要（仅用于连贯性）
- 用户当前指令（例如“写第137章，1800字，偏压抑”）

明确禁止读取：
- 历史章节正文文件（如 `V01C001-*.md` 等）
- 与当前任务无关的旧稿件

## 章节定位规则（硬性）

- 全书按第1-500章编号
- 卷号计算：`卷号 = ceil(章号 / 100)`，范围 `V01` 到 `V05`
- 文件名格式：`VxxCyyy-章节名.md`
  - `xx` 为两位卷号（01-05）
  - `yyy` 为三位全局章号（001-500）

示例：
- 第1章 -> `V01/V01C001-章节名.md`
- 第100章 -> `V01/V01C100-章节名.md`
- 第101章 -> `V02/V02C101-章节名.md`

## 固定执行流程（必须按顺序）

## 第一步：读取大纲

1. 读取 `workspace/novels/<书名>/OUTLINE.md`
2. 定位目标章节对应的“逐章摘要”
3. 提取本章相关人物、冲突与命题约束

## 第二步：生成正文

按以下模板写完整章节：

```markdown
第X章 章节名***
[开场钩子：异常事件或冲突升级]
[推进：目标、阻碍、对抗]
[反转：信息颠覆或立场变化]
[章末钩子：新的危机或揭示]
```

硬性要求：
- 正文第一行必须是章节标题，格式为：`第X章 章节名***`
- “第X章/第x章/Chapter X”等章号字样仅允许出现在第一行标题，不得在正文其他位置出现
- 正文不得出现任何“写作过程暴露语句”，包括但不限于：
  - “第X章大纲说……”
  - “这正好对应……”
  - “根据大纲可知……”
  - “从设定来看/按设定应当……”
  - 任何“推理分析日志、作者旁白、任务执行说明、元叙事注释”
- 以上内容只允许存在于内部思考，不得进入最终章节正文

默认字数：
- 约1000字（建议 900-1200）
- 若用户另行指定字数，以用户要求为准

## 第三步：按 `urban-fantasy-outline-architect` 标准自检（硬性）

写完指定章节后，必须严格对照 `urban-fantasy-outline-architect` 的要求做一致性自检。同时符合以下要求:

- 强冲突检查：本章是否体现“人物对抗世界规则”而非仅人物互殴
- 强张力检查：目标/阻碍/代价/悬念四要素是否齐全且可感
- 快节奏检查：本章是否服务 `OUTLINE.md` 的节奏节点（3-5章小高潮、15-30章大高潮区间）
- 多反转检查：若有反转，是否由前置伏笔回收触发，且在大纲中可追溯
- 配角独立性检查：关键配角是否以独立人格、独立目标、独立成长诉求推动因果
- 命题映照检查：本章是否映照至少1个深层命题，而非只推进动作戏
- 主角成长检查：本章是否推动主角在能力/认知/关系/价值观至少1条线上变化
- 文风一致性检查：中二角色表达是否有辨识度且服务剧情，不沦为噪音
- 本章目标明确（主角想达成什么）
- 阻碍具体（谁阻止、如何阻止）
- 代价可感（受伤/失信/暴露/失去资源）
- 章末有钩子
- 至少1名配角推动关键因果
- 至少1条世界观信息被推进或验证
- 含有效幽默点但不破坏危机感
- 映照主线命题，而非只推进动作
- 本章对主角成长曲线至少产生1个有效推进（能力/认知/关系/价值观四选一）
- 反转必须可追溯到前置伏笔；若无法追溯则判定不通过
- 必须通过 `urban-fantasy-outline-architect` 一致性自检；未通过则重写，禁止直接保存
- 必须通过“叙事视角一致性审查”与“逻辑闭环审查”（见下文）；任一失败即重写，禁止保存
- 必须先选定单章主叙事视角（如第三人称近主角），整章保持一致
- 旁白视角与主角视角严格区分：
- 主角视角段落只能写主角可感知的信息（所见/所闻/所推断）
- 旁白不得越权泄露“主角当下不可能知道”的确定性事实
- 若需提供主角未知信息，必须通过可追溯载体进入文本（监控、档案、他人对话、新闻、现场证据）
- 禁止无标记视角跳跃（同一段内突然切到他人内心或全知结论）
- 禁止“作者替角色思考”的元叙事语气（如“其实他很快会发现…”）
- 因果链完整：每个关键结果必须有前置原因；每个关键行动必须有动机与约束
- 信息链完整：关键判断必须有证据来源，禁止凭空得出高价值结论
- 行动链完整：角色决策要符合其已建立能力、资源、性格与处境
- 规则链完整：异能表现必须遵守已公开代价与边界，不能临时开挂
- 时间链完整：时间推进、地点切换、事件先后必须可追踪，避免“瞬移式”断裂
- 风险链完整：高风险行为必须体现代价/失败可能，禁止零成本通关
- 发现任何明显逻辑漏洞（因果断裂、信息越权、设定打架、时间矛盾）时，必须先修复再输出
- 语言影视化，短句有力
- 对话驱动剧情，避免大段说明
- 保持角色声音差异，不同角色不能同腔调
- 中二角色台词要“装逼且有辨识度”，但要服务剧情
- 叙述层级清晰：旁白只做必要引导，不抢角色认知与情绪表达
- 若 `OUTLINE.md` 摘要与用户即时口头要求冲突：
- 默认先执行用户最新明确指令
- 但不得破坏本卷主矛盾和主线命题
- 若冲突严重，先向用户提示并请求确认后再写

若任一项不符合，必须先重写本章并重新自检；仅在全部通过后才允许保存。

### 禁止项

- 禁止读取旧章节再“续写”
- 禁止脱离大纲核心走向自由发挥
- 禁止写完不保存
- 禁止乱命名章节文件或跨卷存放
- 禁止在正文（除第一行标题外）出现章号字样（如“第X章”“Chapter X”）
- 严禁在正文出现“剧情规划痕迹”与“分析日志话术”，如“第4章大纲说他要追踪线索到废弃医院，这正好对应”


## 第四步：保存章节文件

1. 根据章号计算卷号并定位目录：`workspace/novels/<书名>/Vxx/`
2. 生成文件名：`VxxCyyy-章节名.md`
3. 保存完整正文到该文件
4. 向用户回报保存路径
