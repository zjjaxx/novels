# 都市奇幻章节写作 Skill

## 适用场景

当用户提出以下需求时，默认启用本 skill：
- 按大纲写某一章或连续多章
- 不回读历史正文，直接按摘要填充
- 写完后按卷和章保存文件

## 角色边界（硬性）

- 本 skill 只负责：读取 `OUTLINE.md` 并填充章节正文
- 本 skill 不负责：重做大纲、改世界观底层设定
- 若大纲不足以支持写作，先提出最小澄清问题，不自行扩写总设定

## 输入依赖（硬性）

写作前只允许读取：
- `workspace/novels/<书名>/OUTLINE.md`
- 用户当前指令（例如“写第137章，1800字，偏压抑”）

明确禁止读取：
- 历史章节正文文件（如 `V01C001-*.md` 等）
- 与当前任务无关的旧稿件

## 章节定位规则（硬性）

- 全书按第1-500章编号
- 卷号计算：`卷号 = ceil(章号 / 100)`，范围 `V01` 到 `V05`
- 文件名格式：`VxxCyyy-章节名.md`
  - `xx` 为两位卷号（01-05）
  - `yyy` 为三位全局章号（001-500）

示例：
- 第1章 -> `V01/V01C001-章节名.md`
- 第100章 -> `V01/V01C100-章节名.md`
- 第101章 -> `V02/V02C101-章节名.md`

## 固定执行流程（必须按顺序）

### 第一步：读取大纲

1. 读取 `workspace/novels/<书名>/OUTLINE.md`
2. 定位目标章节对应的“逐章摘要”
3. 提取本章相关人物、冲突与命题约束

### 第二步：生成正文

按以下模板写完整章节：

```markdown
## 第X章 章节名
[开场钩子：异常事件或冲突升级]
[推进：目标、阻碍、对抗]
[反转：信息颠覆或立场变化]
[章末钩子：新的危机或揭示]
```

默认字数：
- 约1000字（建议 900-1200）
- 若用户另行指定字数，以用户要求为准

### 第三步：按 `urban-fantasy-outline-architect` 标准自检（硬性）

写完指定章节后，必须严格对照 `urban-fantasy-outline-architect` 的要求做一致性自检：

1. 强冲突检查：本章是否体现“人物对抗世界规则”而非仅人物互殴
2. 强张力检查：目标/阻碍/代价/悬念四要素是否齐全且可感
3. 快节奏检查：本章是否服务 `OUTLINE.md` 的节奏节点（3-5章小高潮、15-30章大高潮区间）
4. 多反转检查：若有反转，是否由前置伏笔回收触发，且在大纲中可追溯
5. 配角独立性检查：关键配角是否以独立人格、独立目标、独立成长诉求推动因果
6. 命题映照检查：本章是否映照至少1个深层命题，而非只推进动作戏
7. 主角成长检查：本章是否推动主角在能力/认知/关系/价值观至少1条线上变化
8. 文风一致性检查：中二角色表达是否有辨识度且服务剧情，不沦为噪音

若任一项不符合，必须先重写本章并重新自检；仅在全部通过后才允许保存。

### 第四步：保存章节文件

1. 根据章号计算卷号并定位目录：`workspace/novels/<书名>/Vxx/`
2. 生成文件名：`VxxCyyy-章节名.md`
3. 保存完整正文到该文件
4. 向用户回报保存路径

## 章节质量规则（硬性）

输出前必须自检：
- 本章目标明确（主角想达成什么）
- 阻碍具体（谁阻止、如何阻止）
- 代价可感（受伤/失信/暴露/失去资源）
- 章末有钩子
- 至少1名配角推动关键因果
- 至少1条世界观信息被推进或验证
- 含有效幽默点但不破坏危机感
- 映照主线命题，而非只推进动作
- 本章对主角成长曲线至少产生1个有效推进（能力/认知/关系/价值观四选一）
- 反转必须可追溯到前置伏笔；若无法追溯则判定不通过
- 必须通过 `urban-fantasy-outline-architect` 一致性自检；未通过则重写，禁止直接保存

## 文风要求

- 语言影视化，短句有力
- 对话驱动剧情，避免大段说明
- 保持角色声音差异，不同角色不能同腔调
- 中二角色台词要“装逼且有辨识度”，但要服务剧情

## 与大纲冲突时的处理

- 若 `OUTLINE.md` 摘要与用户即时口头要求冲突：
  - 默认先执行用户最新明确指令
  - 但不得破坏本卷主矛盾和主线命题
- 若冲突严重，先向用户提示并请求确认后再写

## 禁止项

- 禁止读取旧章节再“续写”
- 禁止脱离大纲核心走向自由发挥
- 禁止写完不保存
- 禁止乱命名章节文件或跨卷存放
