# 都市奇幻小说存档与更新 Skill

## 适用场景

当用户提出以下需求时，默认启用本 skill：
- 保存 `urban-fantasy-novel-writer` 生成的大纲
- 按章节持续追加、修改或重写小说内容
- 整理卷章结构、统一命名和目录
- 对已有稿件进行版本化更新与变更记录

## 默认存储位置与目录结构

默认根目录：
- `workspace/novels/<书名>/`

标准结构：

```text
workspace/novels/<书名>/
├── 00-大纲.md
├── 01-设定集.md
├── 02-角色档案.md
├── 03-伏笔回收表.md
├── CHANGELOG.md
├── V01/
│   ├── V01C001-章节名.md
│   └── ...
├── V02/
│   └── ...
└── ...
```

## 文件命名规则（硬性）

- 卷目录：`V01` 到 `V05`
- 章节文件：`V01C001-章节名.md`（三位章号，不足补零）
- 若用户未给章节名，使用：`V01C001-未命名章节.md`
- 禁止使用重复文件名；若冲突，先比对内容再决定覆盖或新建

## 固定执行流程（必须按顺序）

### 第一步：识别写入目标

先确认四要素：
- 书名
- 写入类型（大纲/设定/角色档案/章节）
- 卷号与章号（章节场景必填）
- 操作方式（新建/更新/重写）

若用户未给齐信息，按已有内容自动推断；推断失败时再询问用户。

### 第二步：读取现有文件并比对

- 若目标文件存在，先读取后更新，避免误覆盖
- 若为重写章节，保留原文到同目录备份文件：
  - `V01C001-章节名.backup.md`
- 若为小修，直接覆盖目标文件并记录变更摘要

### 第三步：写入正文

写入规范：
- 大纲写入 `00-大纲.md`
- 世界观补充写入 `01-设定集.md`
- 配角和主角资料写入 `02-角色档案.md`
- 伏笔与回收计划写入 `03-伏笔回收表.md`
- 章节写入对应卷目录

章节文件模板：

```markdown
# 第X章 章节名

## 正文
（章节内容）

## 下章预告
（一句预告）
```

### 第四步：更新变更记录

每次写入后必须更新 `CHANGELOG.md`，格式如下：

```markdown
## YYYY-MM-DD HH:mm
- 操作：新建/更新/重写
- 文件：<相对路径>
- 说明：<本次修改摘要>
```

## 更新策略（硬性）

- 增量优先：优先更新指定文件，不全量重写整本书
- 保真优先：除非用户明确要求，不改动已确认章节主情节
- 连续性优先：更新章节时同步检查与前后章设定一致性
- 可追溯优先：任何重大改动必须保留 backup 或 changelog 记录

## 与 `urban-fantasy-novel-writer` 的协作规则

- 将其产出的“大纲、分卷、章节、伏笔表”映射到本 skill 的标准文件
- 如果收到“重写指令”，先备份原稿再写入新稿
- 如果只收到部分文本，按对应文件局部更新，不覆盖无关内容
- 如果章节与设定冲突，先更新设定集并在 changelog 标注冲突修复

## 质量检查清单（写入前后）

- 路径是否正确（是否写入目标书名目录）
- 卷号章号是否正确（是否与文件名一致）
- 章节内容是否约 1000 字（可在 900-1200）
- 是否保留“下章预告”
- 是否更新 `CHANGELOG.md`

## 禁止项

- 禁止直接覆盖整本书目录
- 禁止跳过读取就覆盖已有章节
- 禁止不记录重大改动来源
- 禁止混用命名格式导致后续检索困难
